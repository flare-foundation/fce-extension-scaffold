# [Edge Case] D3-zero-addresses
**Date:** 2026-04-13 14:14:14
**Scenario:** D3-zero-addresses
**Duration:** 0m 0s
**Result:** EXPECTED_FAIL
**Error Clarity:** GOOD

## Expected Behavior
Create a temp addresses file with zero addresses for registries. Run pre-build.sh. Expected: failure. Record error message.

## Actual Behavior
The pre-flight check (Step 0) caught the zero address immediately and exited with code 1.

Error message produced:
```
FATAL  fccutils/encoding.go:29  Error: TeeExtensionRegistry is the zero address (0x0000000000000000000000000000000000000000) in the addresses file
```

Followed by:
```
[pre-build] ERROR: Pre-flight check failed — fix the issues above before deploying
```

The INFO logs before the error showed the zero addresses clearly:
```
INFO  deploy-contract/main.go:35  TeeExtensionRegistry: 0x0000000000000000000000000000000000000000
INFO  deploy-contract/main.go:36  TeeMachineRegistry:   0x0000000000000000000000000000000000000000
```

## Error Message Quality
**GOOD** — The error message is specific and actionable:
1. It identifies exactly which address is wrong ("TeeExtensionRegistry")
2. It names the condition ("is the zero address")
3. It shows the actual value
4. It references the addresses file
5. It fails fast at pre-flight, no gas spent

Note: The error only mentions TeeExtensionRegistry — it stops at the first validation failure and doesn't report that TeeMachineRegistry is also zero. This is acceptable (fail-fast) but could be improved to report all validation errors at once.

## Observations
1. **Distinct error from D1:** D1 (wrong registry / EOA) gets "has no deployed code" while D3 (zero address) gets "is the zero address" — the validation has separate checks for zero vs non-contract addresses. Good differentiation.
2. **Only first error reported:** With both TeeExtensionRegistry and TeeMachineRegistry set to zero, only the first one is reported. A "report all errors" mode would be helpful but is a minor improvement.
3. **Stack trace included:** As with D1, the Go stack trace is included. Consistent behavior.

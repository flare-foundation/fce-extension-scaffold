# [Edge Case] D4-double-deploy
**Date:** 2026-04-13 14:24:20
**Scenario:** D4-double-deploy
**Duration:** 19s (first run) + 16s (second run) = 35s total
**Result:** EXPECTED_FAIL
**Error Clarity:** BAD

## Expected Behavior
Run pre-build.sh twice in a row. Expected: second run creates new contracts/extension. Record: does it warn about existing config/extension.env?

## Actual Behavior
Both runs succeeded with exit code 0. Each run deployed a brand new InstructionSender contract and registered a brand new extension:

- **First run**: Extension ID `...0188`, InstructionSender `0xD5AC7a18109D89840D4342D6b22d6972f2475BBd`
- **Second run**: Extension ID `...0189`, InstructionSender `0x98973F3e70469F7794BA31BA38b1df013D5041C0`

The second run **silently overwrote** `config/extension.env` with the new values. There was:
- No warning about existing extension.env
- No prompt asking if the user wants to overwrite
- No backup of the previous config
- No mention of the previous extension ID

## Error Message Quality
**BAD** — There is no error or warning at all. The script silently creates orphaned on-chain state. A developer who accidentally runs pre-build.sh twice will:
1. Lose track of their previous extension ID
2. Have an orphaned InstructionSender contract and extension registration on-chain
3. Waste gas on the duplicate deployment
4. Have no indication that anything went wrong

## Observations
1. **No idempotency check**: The script should check if `config/extension.env` already exists and either warn, prompt for confirmation, or skip deployment.
2. **Orphaned on-chain state**: Each run creates new contracts that are never cleaned up. On a live network this wastes gas and creates confusion.
3. **Config overwrite without backup**: The `cat > "$CONFIG_OUTPUT"` on line 124 truncates and overwrites without checking if the file exists or creating a `.bak`.
4. **Suggested improvement**: Add a check at the start of pre-build.sh:
   ```bash
   if [[ -f "$CONFIG_OUTPUT" ]]; then
       log "WARNING: $CONFIG_OUTPUT already exists:"
       cat "$CONFIG_OUTPUT"
       log "Re-running will deploy NEW contracts and overwrite this config."
       log "Use --force to proceed, or delete $CONFIG_OUTPUT first."
       exit 1
   fi
   ```
